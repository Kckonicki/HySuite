Permissions Reference

Complete list of all HySuite permission nodes for fine-grained access control.

Note: All permissions use the hysuite. prefix. Player commands (TPA, homes, warps, spawn) don't require permissions by default.

Quick Setup Examples

Give Player Full Admin

/lp user <name> permission set hysuite.admin.* true
/lp user <name> permission set hysuite.fly true
/lp user <name> permission set hysuite.vanish true
/lp user <name> permission set hysuite.setwarp true
/lp user <name> permission set hysuite.delwarp true

Create VIP Group

/lp creategroup VIP
/lp group VIP permission set hysuite.vip true
/lp group VIP permission set hysuite.vip.homes true
/lp group VIP permission set hysuite.vip.cooldown true
/lp group VIP permission set hysuite.cooldown.bypass true

Create Moderator Group

/lp creategroup Moderator
/lp group Moderator permission set hysuite.adminchat.staff true
/lp group Moderator permission set hysuite.vanish true
/lp group Moderator permission set hysuite.htp true
/lp group Moderator permission set hysuite.htphere true

Admin Permissions

Permission Description Default
hysuite.admin.ranks Access rank management GUI OP
hysuite.admin.setrank Set or remove player ranks OP
hysuite.admin.playerinfo View detailed player information OP
hysuite.admin.reload Reload plugin configuration OP

Communication Permissions

Permission Description Default
hysuite.adminchat.staff Access staff chat channel OP
hysuite.adminchat.admin Access admin chat channel OP

Chat Channel Details

  • Staff Chat - For moderators and staff members
  • Admin Chat - For administrators only (higher level)
  • Toggle with /adminchat [staff|admin]

Utility Permissions

Permission Description Default
hysuite.vanish Toggle vanish mode (invisible to players) OP
hysuite.fly Toggle fly mode OP
hysuite.htp Teleport to other players OP
hysuite.htphere Teleport players to you OP

Warp Management Permissions

Permission Description Default
hysuite.setwarp Create server warps OP
hysuite.delwarp Delete server warps OP
hysuite.setspawn Set server spawn point OP
Note: Players don't need permissions to use warps (/warp, /warps) or spawn (/spawn).

VIP Permissions

Permission Description Benefit
hysuite.vip VIP status marker Recognition, special features
hysuite.vip.homes Increased home slots 10 homes instead of 3 (configurable)
hysuite.vip.cooldown Reduced cooldown times 50% faster cooldowns (configurable)
hysuite.cooldown.bypass Bypass all cooldowns & warmups Instant teleports, no delays

VIP Configuration

Customize VIP benefits in config.json:

{
  "maxHomes": 3,           // Default players
  "vipMaxHomes": 10,       // VIP players
  "teleportCooldown": 30,  // Default cooldown
  "vipCooldownReduction": 0.5  // 50% reduction for VIPs
}

Rank-Based Permissions

Permission Pattern Description Example
hysuite.rank.* Wildcard for all rank permissions Admin groups
hysuite.rank.<rankid> Specific rank assignment hysuite.rank.moderator
Important: Ranks are managed through HySuite's internal system. These permissions are auto-assigned when using the rank GUI.

Wildcard Permissions

Wildcard Grants Use Case
hysuite.* All HySuite permissions Owner/Full Admin
hysuite.admin.* All admin commands Server Admins
hysuite.adminchat.* All chat channels Staff members
hysuite.vip.* All VIP perks VIP donors
Warning: Be careful with wildcards! hysuite.* grants ALL permissions including destructive commands.

No Permission Needed

These commands work for all players by default:

  • All TPA commands (/tpa, /tpahere, /tpaccept, /tpdeny, /tpcancel)
  • All home commands (/sethome, /home, /delhome, /homes)
  • Warp usage (/warp, /warps)
  • Spawn teleport (/spawn)
  • Utility (/back, /rtp)
  • Messaging (/msg, /reply)

Permission Plugin Examples

LuckPerms

# Create group
/lp creategroup VIP

# Add permissions
/lp group VIP permission set hysuite.vip true
/lp group VIP permission set hysuite.cooldown.bypass true

# Add player to group
/lp user <name> parent add VIP

# Remove permission
/lp user <name> permission unset hysuite.fly

PermissionsEx (PEX)

# Add permission to group
/pex group VIP add hysuite.vip

# Add permission to user
/pex user <name> add hysuite.fly

# Set group
/pex user <name> group set VIP

Testing Permissions

Check if Permission Works

# LuckPerms
/lp user <name> permission check hysuite.fly

# PermissionsEx
/pex user <name> has hysuite.fly

Debug Permission Issues

  1. Verify permission is spelled correctly (hysuite. prefix)
  2. Check player's group has the permission
  3. Reload permissions: /lp sync or /pex reload
  4. Check for negated permissions (- prefix)
  5. Verify player is in correct group
